    Difference between objects:

      Literal Objects:
    1. Has no constructors
    2. Cannot be inherited
    3. Best used for “one off” objects where only one copy will exist
    4. A colon separates the key/value pair
    5. For global or configuration objects like 'game settings
    1. Has a constructor
    2. Can be inherited (but very messy, out of scope here)
    3. Best use for stuff that you want to instantiate again and again
    4. The this keyword is used instead of the object name
    5. If you have lots of objects with similar functionality
